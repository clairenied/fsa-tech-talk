Referred to as true entropy

Blocking: slow because one has to go out there and harvest some entropy

Lots of people use radioactive decay for this (in the short term, the time between two decay events is random), but as we all know that is wholly impractical for the purposes of this tech talk

Also, it makes me nervous

Ok, so let's do that!

With HotBits!

The friendly HotBits people are out there measuring time between radioactive events! Just for us!

And then serving it up on their website!

Ok but what if we want to do this in a completely closed system where you don't have to talk to anyone

AKA The nerd's dream

This seed would be the initial source for the algorithm to do its thing

Unlike true entropy, this is generally considered to be pseudorandom

It can still be seeded with an entropic event from nature

But since we don't have to wait for that event to occur, to store up the data, it's non-blocking and potentially a whole lot faster!

Because I am not making anything that needs to be cryptographically sound, this should do:

The variance of a uniformly distributed random variable on the inteval of -0.5 to 0.5 is 1/12

The sum of 12 random variables with mean 0 and variance 1/12 is a random variable with mean 0 and variance 1

Adding up a bunch of independent, identically distributed random variables will give you an approximate Gaussian random variable

Which is good enough for me!
